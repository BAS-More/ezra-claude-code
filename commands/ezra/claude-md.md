---
name: ezra:claude-md
description: Generate or update CLAUDE.md from EZRA governance state so every Claude Code session starts with full context.
---

You are executing the EZRA CLAUDE.md Generator. This command creates or updates a `CLAUDE.md` file at the project root using data from `.ezra/` state.

## Phase 1: Read EZRA State

Read the following files (skip any that don't exist):

1. **`.ezra/governance.yaml`** — project name, language, framework, protected paths, standards, enforcement config
2. **`.ezra/knowledge.yaml`** — architecture, patterns, layers, dependencies, entry points
3. **`.ezra/decisions/`** — all decision files; filter to only `status: ACTIVE` (exclude SUPERSEDED, DEPRECATED)
4. **`.ezra/versions/current.yaml`** — current version, health score
5. **`package.json`** — project name, version, dependencies (if exists)
6. **`tsconfig.json`** — TypeScript config (if exists)
7. **`pyproject.toml`** — Python config (if exists)

If `.ezra/` does not exist at all, display:
```
EZRA: Cannot generate CLAUDE.md — .ezra/ not initialized.
Run /ezra:init or /ezra:bootstrap first.
```
And stop.

## Phase 2: Generate Content

Build the CLAUDE.md content with these sections:

```markdown
<!-- EZRA:START — Auto-generated by /ezra:claude-md. Do not edit between markers. -->

# Project: {name}

## Architecture
{From knowledge.yaml — describe layers, patterns, key directories}
{Example: "3-tier architecture: Routes -> Services -> Repositories. NestJS framework with TypeScript."}

## Active Decisions
{List each ACTIVE decision from .ezra/decisions/}
{Format: "- **ADR-NNN**: {decision text} [{category}] — Enforced on: {affected_paths}"}

## Protected Paths
{From governance.yaml protected_paths}
{Format: "- `{pattern}` — {reason}"}

## Standards
{From governance.yaml standards section}
{List each standard with its value}

## Tech Stack
{From package.json/tsconfig/pyproject — language, framework, key dependencies}

## Governance
This project uses EZRA for codebase governance.
- Run `/ezra:status` for current health
- Run `/ezra:scan` before major changes
- Run `/ezra:guard` before committing
- All architectural decisions must be recorded via `/ezra:decide`
- Current health score: {score}/100
- EZRA version: {version}

<!-- EZRA:END -->
```

## Phase 3: Write or Update CLAUDE.md

**If CLAUDE.md does NOT exist:**
- Create it with the full generated content
- The entire file content is the EZRA-managed block

**If CLAUDE.md ALREADY exists:**
- Read the existing content
- Look for `<!-- EZRA:START` and `<!-- EZRA:END -->` markers
- If markers exist: replace ONLY the content between markers (inclusive of markers) with the new generated content. Preserve everything before `<!-- EZRA:START` and after `<!-- EZRA:END -->`.
- If markers do NOT exist: append the generated content (with markers) to the END of the existing file, preceded by two blank lines

**Important:** Never delete or modify content outside the EZRA markers. The user may have custom sections in their CLAUDE.md.

## Phase 4: Report

Display:
```
EZRA CLAUDE.md Generated
═══════════════════════════════
Project: {name}
Sections: Architecture, {N} Active Decisions, {N} Protected Paths, Standards, Tech Stack, Governance
Mode: {Created | Updated (preserved existing content)}

The CLAUDE.md file ensures every Claude Code session starts
with full governance context.
```

## Edge Cases

- If no active decisions exist, write "No active decisions recorded yet. Use `/ezra:decide` to record decisions."
- If no protected paths configured, write "No protected paths configured yet."
- If knowledge.yaml is missing or empty, write a minimal architecture section noting it needs a scan: "Architecture not yet scanned. Run `/ezra:scan` for analysis."
- If health score is not available, write "Health score: Not assessed. Run `/ezra:health`."

Do NOT ask for confirmation. Execute all phases automatically.
